# essence

> 复杂表象之下，有一个极简的本质。  
> 这个仓库是我主动去触碰它的地方。

---

## 为什么存在

小时候我喜欢用 4 去数 5 个物体。

不是在计算，是在等待。等待两个节奏在某一刻恰好对齐。那种对齐的瞬间有一种说不清楚的满足感——不是"我算对了"，而是"我感受到了某种真实的东西"。

后来我知道这叫最小公倍数。但数学课只告诉你答案是 20，它没有办法给你那种一步一步走到 20 的感觉。

北美有一种蝉，13 年钻出地面一次。另一种，17 年一次。两个都是质数，所以它们同时出现要等 221 年。它们不知道自己在用数论，但自然界把这个结构演化出来了。我第一次听到这件事的时候，感受到的是和数瓷砖完全一样的东西。

这就是这个仓库想做的事：

**找到那些藏在表象下面的本质，然后用代码把它触碰一遍。**

不是为了学习，不是为了面试，是为了那种"原来世界是这样运作的"的感觉。

---

## 结构

```
essence/
  light/      # 光的本质——光线追踪，从零开始
  motion/     # 力的本质——物理引擎，重力、碰撞、轨道
  wave/       # 频率的本质——傅里叶，任何周期性的东西
  rhythm/     # 数与节奏——对齐、质数、最小公倍数
```

每个模块不追求完整，追求**最小可以让人感受到本质的实现**。

---

## 原则

- 不用库解决能自己实现的东西
- 实现之前先问：这个东西最底层的形状是什么
- 实现之后先问：我现在理解了什么是我之前没有的

---

## 语言

Rust。因为它逼你直面机器真实的样子，没有东西替你藏着。
